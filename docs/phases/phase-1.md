# PHASE 1: FOUNDATION (2026–2028)
### "You cannot build what you don't understand at the atomic level."

## OBJECTIVE
Learn the primitives. No building yet. Pure acquisition of leverage.

## WHAT TO EXPECT
- Discomfort. Math will feel foreign. Push through.
- First 6 months: everything feels theoretical. That's correct.
- By month 18: you will see the connections between cryptography, film, and capital.
- By month 24: you can read ZK proof papers without a dictionary.

## LEARNING TRACKS

### 1. Mathematics
**Why:** The substrate of cryptography and ML. You cannot reason about ZK proofs without it.
- [ ] Linear Algebra
- [ ] Probability & Statistics
- [ ] Discrete Mathematics & Logic
- [ ] Number Theory (basic)

**Resources:**
- *Mathematics for Machine Learning* — Deisenroth (free PDF: mml-book.github.io)
- 3Blue1Brown — Essence of Linear Algebra (YouTube, free)
- Khan Academy — Discrete Math (free)
- *An Introduction to Mathematical Cryptography* — Hoffstein et al.

**Milestone:** Understand why elliptic curves are used in cryptography.

---

### 2. Cryptography
**Why:** ZK proofs, hardware attestation, and blockchain are all applied cryptography.
- [ ] Hash functions (SHA-256, Poseidon)
- [ ] Public/private key infrastructure
- [ ] Digital signatures
- [ ] What a zero-knowledge proof is at the logic level
- [ ] Merkle trees

**Resources:**
- Stanford Cryptography I & II — Dan Boneh (Coursera, free audit)
- *Serious Cryptography* — Jean-Philippe Aumasson (book)
- ZKProof.org community standards documents (free)

**Milestone:** Explain to someone how a ZK proof lets you prove knowledge without revealing it.

---

### 3. Rust Programming
**Why:** Memory safety without garbage collection. Essential for hardware-level systems and cryptographic code.
- [ ] Ownership and borrowing
- [ ] Lifetimes
- [ ] Error handling (Result, Option)
- [ ] Traits and generics
- [ ] Async Rust basics
- [ ] Build a CLI tool
- [ ] Build a simple TCP server

**Resources:**
- *The Rust Book* (official, free) — doc.rust-lang.org/book
- Rustlings exercises (free) — github.com/rust-lang/rustlings
- *Programming Rust* — Blandy & Orendorff (book, deep dive)
- Jon Gjengset YouTube channel — advanced Rust livestreams

**Milestone:** Build a file integrity checker in Rust that hashes documents and detects tampering.

---

### 4. Filmmaking as Protocol
**Why:** The film is the proof of concept. Every production is a live demo of the stack.
- [ ] Study narrative structure neurologically (not as formula)
- [ ] Study Roger Deakins — every lighting decision as language
- [ ] Study Christopher Nolan's non-linear structure mechanics
- [ ] Shoot one short film. Treat footage as provenance artifact.
- [ ] Write one complete screenplay

**Resources:**
- *Story* — Robert McKee (book)
- *The Conversations* — Michael Ondaatje & Walter Murch (book)
- Roger Deakins' online masterclass
- *On Filmmaking* — Alexander Mackendrick (book)

**Milestone:** Complete short film. Register it. Document the provenance chain manually (as practice for Canonchain).
